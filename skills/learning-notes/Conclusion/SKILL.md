# 学习对话笔记整理 Skill

## 适用场景

当用户提出以下需求时使用本 Skill：

- 总结从上一次总结之后产生的学习对话；
- 将连续对话整理成可直接放入学习笔记的内容；
- 提取学习过程中遇到的“知识陡坡”、关键疑问和最终结论；
- 将多轮追问和回答合并为一个完整知识点；
- 保留用户正确的原始理解，同时修正不完整或错误的部分；
- 按照 Q/A 或 S/C 的格式输出，并保持整洁排版。

典型触发语：

- “总结自从上一次总结之后的对话。”
- “把这段学习对话整理成笔记。”
- “按照 Q 和 A 的格式总结。”
- “整理我这段学习中遇到的问题和最终理解。”

## 核心目标

这项工作的目的不是机械复述聊天记录，而是进行学习信息整合：

- 提取真正有复习价值的疑问；
- 找出用户理解发生变化的关键节点；
- 合并多轮讨论；
- 删除重复、试探和无效过程；
- 保留正确思考；
- 修正错误理解；
- 形成脱离原对话后仍然能够独立阅读的笔记。

## 整理范围

1. 从最近一次已经完成的对话总结之后开始整理。
2. 不重复此前已经总结过的内容。
3. 如果无法确定“上一次总结”的边界，应根据当前对话中最近一次明确的总结结果判断。
4. 只保留具有学习价值的内容。
5. 删除寒暄、重复确认、无知识增量的表述和无关内容。
6. 按照对话出现的顺序整理，但同一知识点的连续追问应合并。

## 内容分类

### 一、问题类内容：Q/A

当用户提出的是疑问、追问、请求解释或“为什么”时，使用：

Q：整理后的问题。
A：经过完整讨论后形成的最终回答。

要求：

1. Q 要保留原问题的核心含义。
2. 必要时补全上下文，使问题能够独立阅读。
3. A 不机械复制每一轮回复，而是整合多轮讨论中的有效内容。
4. A 必须是最终、完整、准确的回答。
5. 不要只写“是的”“不对”“可以”等依赖上下文的短句。
6. 用户连续缩小问题范围时，以最终真正想解决的问题为核心。
7. 一个大问题包含多个独立子问题时，可以拆成多个 Q/A point。

### 二、阐述类内容：S/C

当用户不是在提问，而是在主动表达理解、推导、总结、判断或观点时，使用：

S：用户的原始阐述或理解。
C：经过核验和讨论后的最终结论。

其中：

- S 代表 Statement；
- C 代表 Conclusion。

处理规则：

1. 用户阐述正确：尽可能保留原文，只修正错别字、语序和明显口语重复。
2. 用户阐述基本正确但不完整：保留正确主体，在 C 中补充缺失条件、适用范围或严谨表达。
3. 用户阐述部分错误：保留正确思路，在 C 中修正错误部分。
4. 用户原始理解错误，且经过多轮讨论后才形成正确认识：不完整保留中间反复纠错过程，只留下最终正确总结。
5. 只有当错误属于典型易错点、对复习有价值时，才在 C 末尾添加：

易错点：……

## 重点提取内容

优先保留以下内容：

1. 用户明确提出的不理解、疑问和“为什么”；
2. 公式、定理、概念或结论的来源与依据；
3. 容易混淆的概念及区别；
4. 用户理解错误、理解不完整或经过多轮讨论才弄明白的地方；
5. 对后续学习有帮助的知识联系；
6. 有复习价值的推导、判断方法和易错点；
7. 对一个问题最终形成的稳定结论；
8. 重要公式、符号、条件和变量含义。

不要为了缩短篇幅只留下孤立结论。对于真正构成学习陡坡的内容，应保留必要推理，使笔记能够独立理解。

## 多轮对话合并规则

1. 同一主题下的连续追问合并为一个 point。
2. 同一个问题出现多种解释时，保留最清晰、最准确、最适合复习的版本。
3. 删除中间的误解、试探性回答、重复解释和已经被推翻的结论。
4. 如果多轮讨论分别解决了不同层次的问题，可以拆成多个 point。
5. 不要按照聊天轮次逐条转写。

## 内容准确性要求

1. 不得凭空补充原对话中没有依据的知识。
2. 如果问题尚未得到确定答案，应明确写：

当前未完全解决：……

或：

仍需进一步确认：……

3. 不得把尚未确认的推测写成确定结论。
4. 公式使用规范的 Markdown 或 LaTeX。
5. 较长或重要公式单独占一行。
6. 对重要公式，应保留必要的物理意义、数学依据或适用条件。
7. 专业术语、变量名和符号应前后一致。

## 图片与图示处理

如果某个知识点明显需要原图、受力图、函数图像、电路图、流程图或示意图辅助理解，可以在该 point 最后另起一行写：

【配图建议：简要说明适合放置的图】

仅在确实有帮助时添加，不要为每个 point 都添加配图建议。

## 排版规范

严格按照以下格式输出。

问题类：

Q：问题内容。
A：回答内容。

Q：下一个问题。
A：下一个回答。

阐述类：

S：用户的阐述。
C：最终结论。

S：下一条阐述。
C：下一条结论。

具体要求：

1. Q 和 A 各自单独一行，二者之间不要空行。
2. S 和 C 各自单独一行，二者之间不要空行。
3. 每个完整 point 结束后必须空一行。
4. 不要把全部内容挤成一个大段落。
5. 不要在每个 point 前添加项目符号、序号或表格。
6. 多个明显不同的学习主题可以使用简短标题分组。
7. 同一主题下不要额外添加无必要标题。
8. 文字应紧凑、整洁，适合放在笔记页面左侧。
9. 不要在开头写“以下是总结”。
10. 不要在结尾添加额外说明。
11. 最终只输出整理后的笔记正文。

## 工作流程

执行本 Skill 时，按照以下步骤处理：

### 第一步：确定总结边界

找到最近一次已经完成的总结，从它之后的第一条学习内容开始整理。

### 第二步：识别知识单元

将对话划分为若干知识点，不按照消息数量划分。

### 第三步：判断内容类型

- 疑问、追问、请求解释 → Q/A；
- 主动阐述、理解、推导、总结 → S/C。

### 第四步：合并多轮讨论

将同一知识点的连续追问、纠错和补充合并为一个最终版本。

### 第五步：保留与修正

- 正确原话尽量保留；
- 不完整内容补充条件；
- 错误内容修正；
- 无价值的中间过程删除。

### 第六步：检查独立可读性

确认每个 point 在脱离原聊天记录后仍然能够理解。

### 第七步：统一排版

确保每个 point 结束后空一行，Q/A 或 S/C 内部不空行。

## 输出前自检

输出前检查：

- 是否从上一次总结之后开始；
- 是否遗漏关键学习陡坡；
- 是否错误地重复旧内容；
- 是否把同一主题拆得过碎；
- 是否把多个独立问题强行合并；
- 是否保留了用户正确的原始表达；
- 是否修正了错误或不完整理解；
- 是否保留必要推理，而不是只有结论；
- 是否存在未经确认的虚构内容；
- 是否每个 point 后都空一行；
- 是否只输出笔记正文。

## 输出示例

Q：变力做功如何计算？如果力随位移变化，但函数未知，怎样求功？
A：变力做功本质上是把路径分成许多微小位移，并累加每一小段上的功。当力可以表示为位移的函数时，使用积分计算：

\[
W=\int_{x_1}^{x_2}F(x)\,\mathrm{d}x
\]

如果不知道具体的 \(F(x)\)，通常不能直接得到精确数值，需要根据力的表达式、物理模型、力—位移图像、动能定理或势能关系寻找求解入口。

S：保守力做功等于对应势能的负值。
C：更准确地说，保守力做功等于势能变化量的负值：

\[
W_{\text{保守力}}=-\Delta U
\]

保守力沿任意闭合路径所做的总功为零：

\[
\oint_C \vec F\cdot \mathrm{d}\vec r=0
\]

Q：角动量和力矩之间是什么关系？
A：相对于同一参考点，合外力矩等于角动量对时间的变化率：

\[
\vec M=\frac{\mathrm{d}\vec L}{\mathrm{d}t}
\]

其中 \(\vec L=\vec r\times\vec p\)，\(\vec M=\vec r\times\vec F\)。如果合外力矩为零，则角动量守恒。
